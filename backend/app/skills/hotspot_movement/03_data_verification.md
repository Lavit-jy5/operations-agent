# Skill：数据核验

## 目标

防止不可核验、来源不明或口径混乱的数据进入标题和机会解读。

## 数据使用红线

任何进入标题或解读的精确数字，必须至少满足一项：

1. 用户材料中明确给出来源名和发布日期。
2. 来自官方、交易所、上市公司公告、权威财经媒体、行业协会、知名机构报告。
3. 来自用户明确声明的 Wind/Excel/内部数据，并给出日期和口径。

如果不满足：

- 禁止写入标题。
- 禁止写成精确数字。
- 改用定性表达。

示例：

- 不写：「A股汽车零部件+4.04%、12只涨停」
- 改写：「A股汽车零部件逆势领涨、多股涨停」

## 必查字段

每条核心数据尽量具备：

- 来源名
- 发布日期或数据日期
- 指标口径
- 单位
- 适用范围

缺失时，在 `review_issues` 中提示。

## 优先来源

1. 官方数据：央行、统计局、交易所、行业协会。
2. 权威媒体：财联社、中证报、证券时报、上证报、第一财经、路透、彭博、CNBC、新华社。
3. 专业机构：中信、中金、华泰、瑞银、大摩、高盛、世界黄金协会、CRU、IDC、Yole。
4. 用户提供的 Wind/Excel 数据。

## 禁用来源

- 无署名自媒体。
- 股吧/论坛帖子。
- 搬运截图。
- 未标注来源的长图。
- 用户未确认的小道消息。

这类信息只能作为方向性参考，不能作为硬数据支撑。

## 自查输出

生成结果中如发现问题，应在 `review_issues` 标注：

- `warning`：数据可用但缺口径、单位或日期。
- `error`：数据不可核验却被用于关键结论。
- `info`：建议补充来源或使用更强数据锚点。
